# 🛡️ Endpoint Security & Monitoring: Deploying Wazuh SIEM for Real-Time Threat Detection

## 📌 Project Overview
In this hands-on cybersecurity lab, I deployed and configured **Wazuh**, an open-source Enterprise Security monitoring solution. The goal was to establish a centralized **Security Operations Center (SOC)** environment to monitor a Windows 11 endpoint in real-time. 

This project demonstrates practical expertise in:
* **Real-time File Integrity Monitoring (FIM):** Successfully detected and alerted on unauthorized file creations and deletions within sensitive directories.
* **Log Analysis & Authentication Monitoring:** Tracked user authentication patterns, specifically identifying failed logon attempts to monitor for potential unauthorized access.
* **SIEM Dashboards & Custom Filtering:** Utilized **DQL (Dashboards Query Language)** to reduce alert noise and isolate high-priority security threats (Level 7+) for efficient incident response.
---

## 🛠️ Environment & Infrastructure Setup
The lab was built using **Oracle VirtualBox** with a **Bridged Networking** configuration. This was a critical step to ensure seamless communication between the SIEM manager (VM) and the monitored endpoint (Host).

* **SIEM Manager:** Ubuntu Server 22.04 (Running as a VirtualBox VM)
* **Monitored Endpoint:** Windows 11 (Physical Host Machine)
* **Network Mode:** Bridged Adapter (Allows both machines to be on the same subnet)
* **Security Stack:** Wazuh Manager (Server-side), Wazuh Agent (Endpoint-side), DQL (Dashboards Query Language for data analysis)

## ⚙️ Phase 1: Wazuh Manager Deployment (Ubuntu Server)
I initiated the Wazuh manager installation on a dedicated Ubuntu server. The installation followed a structured process to ensure security and integrity.

### 🚀 Installation Steps:
1. **GPG Key Integration:** Added the official Wazuh GPG key to verify package authenticity:
   `curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo gpg --dearmor -o /usr/share/keyrings/wazuh-archive-keyring.gpg`

2. **Automated Installation:** Executed the installation script to deploy the indexer, server, and dashboard:
   `curl -sO https://packages.wazuh.com/4.12/wazuh-install.sh && sudo bash ./wazuh-install.sh -a -i`

3. **Network Validation:** Identified the manager’s IP (**172.20.10.11**) for agent communication.

3. **Post-Installation Validation:**
After the process was completed, I identified the Manager's IP as **172.20.10.11**, which was essential for registering the Windows agent in the next phase.

## 🛡️ Phase 2: Agent Onboarding via PowerShell (Administrative Setup)

To monitor the Windows host, I deployed the Wazuh Agent using PowerShell. This approach ensures a faster and more consistent deployment across endpoints.

### 🔑 Prerequisites: Elevated Privileges
Before running any commands, I launched **PowerShell as an Administrator** to ensure the necessary permissions for installing the agent and managing system services.

### 🚀 Deployment & Verification Steps:

1. **Manager Connectivity Check:**
I first verified that the Wazuh Manager was reachable and the API status was "Online" through the Wazuh Dashboard.

2. **Deploy Agent and installation:**
To monitor the Windows host, I deployed the Wazuh Agent using the customized deployment script generated by the Wazuh Dashboard. This ensures the agent is pre-configured to communicate with the manager.

### 🔑 Step 1: Generating the Deployment Command
I navigated to the **"Deploy new agent"** section in the Wazuh Dashboard. I configured the following settings to generate a unique installation command:
* **Operating System:** Windows
* **Wazuh Server Address:** 172.20.10.11 (The Ubuntu Manager's IP)
* **Agent Name:** My-Windows-PC

### 🚀 Step 2: Automated Installation & Enrollment
I launched **PowerShell as an Administrator** and executed the generated command. This single-line script automates the download, silent installation, and registration of the agent with the manager.

**The full command executed was:**
```powershell
Invoke-WebRequest -Uri [https://packages.wazuh.com/4.x/windows/wazuh-agent-4.12.0-1.msi](https://packages.wazuh.com/4.x/windows/wazuh-agent-4.12.0-1.msi) -OutFile ${env:tmp}\wazuh-agent.msi; msiexec.exe /i ${env:tmp}\wazuh-agent.msi /q WAZUH_MANAGER='172.20.10.11' WAZUH_REGISTRATION_SERVER='172.20.10.11' WAZUH_AGENT_NAME='My-Windows-PC'

3. **Starting the Wazuh Service:**
Once installed, I started the agent service manually to begin the registration process:

NET START WazuhSvc

Result: The terminal output confirmed: "The Wazuh service was started successfully".

## 🛑 Phase 3: Real-Time Threat Detection & Security Analysis

### 1. File Integrity Monitoring (FIM)
I configured the Wazuh agent to monitor the `C:\test` directory for any unauthorized file modifications.
* **Activity:** Created and deleted a file named `hacker.txt`.
* **Detection:** As shown in the dashboard, Wazuh captured events for **"File added to the system"** and **"File deleted from the system"**.
* **Alert Level:** The deletion was flagged as a **Level 7 Alert**, demonstrating the system's ability to detect high-priority integrity changes.

## 💡 Troubleshooting & Challenges Faced

* **Network Isolation:** Initially, the Windows Agent could not communicate with the Ubuntu Manager. I resolved this by reconfiguring the VirtualBox Network settings from **NAT** to a **Bridged Adapter**, placing both machines on the same subnet.
* **Alert Noise Reduction (DQL Filtering):** The dashboard was initially overwhelmed with low-level system logs. 
* **Solution:** I implemented a custom **DQL (Dashboards Query Language)** filter: 
  `rule.level >= 7 and not rule.groups: "sca"`
  This successfully filtered out background "noise" and isolated high-priority security threats, allowing for more efficient analysis.
